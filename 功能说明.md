# 日记本 - 功能说明

## crypto.h / crypto.cpp —— 加密引擎

直接复用密码库（MiMaKu_Code）的加密代码，仅修改了 MAGIC 标识。

### 常量

| 常量 | 值 | 说明 |
|------|-----|------|
| `MAGIC` | `0x31524944` | `DIR1` little-endian，日记文件标识 |
| `VERSION` | `1` | 文件格式版本号 |
| `HEADER_SIZE` | `48` | `magic + version + salt + nonce` |
| `SALT_LEN` | `16` | Argon2id 盐长度 |
| `NONCE_LEN` | `24` | XChaCha20 Nonce 长度 |
| `KEY_LEN` | `32` | 派生密钥长度 |
| `TAG_LEN` | `16` | 认证标签长度 |

### `bool init()`

调用 `sodium_init()`，初始化 libsodium。

### `vector<unsigned char> encryptFile(plaintext, password)`

流程：

1. 生成随机 Salt + Nonce
2. 用 Argon2id 从密码派生密钥
3. 用 XChaCha20-Poly1305 加密
4. 组装二进制头 + 密文
5. 擦除密钥内存

### `optional<string> decryptFile(data, size, password)`

流程：

1. 校验 magic/version
2. 提取 Salt / Nonce
3. 派生密钥
4. 解密并验证认证标签
5. 成功返回 JSON 字符串，失败返回 `nullopt`

## diary_store.h / diary_store.cpp —— 日记数据管理

负责 JSON 数据、按日期 CRUD、神秘计数器与历史时间链、加密读写。

### JSON 数据结构

```json
{
  "version": 1,
  "entries": [
    {
      "year": 2026,
      "month": 5,
      "day": 1,
      "segments": [
        {"time": "16:56", "content": "早上写的日记"},
        {"time": "19:45", "content": "晚上补充的"}
      ]
    }
  ],
  "counter": 0,
  "counter_history": []
}
```

- `entries`：按天存日记
- `segments`：同一天内多条时间段记录
- `counter`：神秘计数器当前值
- `counter_history`：最近几次触发计数器的时间戳数组
  - 仍然只存时间戳，不单独存编号
  - 计数器页面右侧显示的 `#编号` 由当前 `counter` 在渲染时倒推得到

### 核心接口

| 接口 | 说明 |
|------|------|
| `load(filePath, password)` | 解密 `.enc` 并解析 JSON |
| `save(filePath, password) const` | JSON 序列化后重新加密写回 |
| `initEmpty()` | 初始化空白库，含 `counter` 与 `counter_history` |
| `findEntry(year, month, day)` | 按日期找条目 |
| `addEntry(entry)` | 添加新条目 |
| `removeEntry(index)` | 删除条目 |
| `updateEntry(index, entry)` | 替换条目 |
| `getSortedIndices()` | 返回按日期升序的索引列表 |
| `getCounter()` / `setCounter()` | 读取 / 修改计数器 |
| `getCounterHistory()` | 读取最近触发时间链 |
| `pushCounterHistory(timestamp, maxCount)` | 追加一条时间戳，并裁剪上限 |
| `trimCounterHistoryTail(removeCount)` | 从历史尾部删掉最近几条触发记录 |

## 写入 / 编辑今日 —— `writeOrEditToday(store, password)`

这是日记本最核心的交互。

### 当前行为

1. 获取当前日期和时间
2. 查找今日日记
3. 如果没有则新建今天的 entry
4. 如果已有则给今天追加一个空 segment
5. 组装 `TODAY_LOG`
6. 打开多行编辑器 `openDiaryEditor()`
7. 用户确认后保存；取消则回滚

### 编辑器特征

- 上方：`TODAY_LOG`
- 下方：`INPUT`
- 当前输入会实时投影到 `TODAY_LOG` 末尾
- `PgUp/PgDn` 翻历史页
- `Alt+Up/Down` 细滚动历史视图
- `Up/Down` 保留给编辑光标移动，不与阅读页统一
- 中文输入法下的弯引号 `“ ” ‘ ’` 已按单宽字符处理，不再出现“实际输入一个字符、光标视觉多空一格”的错位

## 按日期编辑 —— `editByDate(store, password)`

现在已经不是旧的“年→月→日纯文本跳屏菜单”了，而是两页式：

1. `DATE.SELECTOR`
2. `DATE.ENTRY`

### `DATE.SELECTOR`

用途：

- 选择年份
- 选择月份
- 选择日期

当前行为：

- 左侧列表区 + 右侧 context 区
- 顶部只保留 `PATH` / `MODE`
- 底部灰色提示：`Enter进入  Esc返回  PgUp/PgDn翻页  Home/End首尾`
- 底部状态：`>> DATE INDEX READY`
- 使用 `menuSelectScrollableInRegion()` 支持长列表滚动
- 常规层级切换只做局部刷新，不再每次整页重画

### `DATE.ENTRY`

用途：

- 展示某一天的所有 segment 预览
- 直接编辑 / 添加 / 删除 segment

页面结构：

- 上半区：`[ ENTRY_LOG ]`
- 下半区：`[ SEGMENT_ACTIONS ]`

底部：

- 灰色提示：`Enter执行  Esc返回  PgUp/PgDn翻页`
- 状态：`>> ENTRY OPERATIONS READY`

### 当前支持的操作

| 操作 | 说明 |
|------|------|
| 编辑记录 N | 打开编辑器修改指定 segment |
| 添加新记录 | 自动补当前时间，进入编辑器 |
| 删除某条记录 | 通过 `SEGMENT.SELECT` 先选记录，再确认删除 |
| 删除整篇日记 | yes 确认后删除当天 entry |
| 返回日期索引 | 返回 `DATE.SELECTOR` |

### 闪屏策略

- `DATE.ENTRY` 不再每轮循环整页重画
- 首次进入时仍会首绘一次壳子
- 常规菜单移动时只更新头部、预览区和操作区
- resize 或视口变化时才重新首绘

## 导出 —— `exportDiary(store)`

导出到：`data/export_diary.txt`

导出格式仍保持：

```text
————————————————————————
2026/3/30

17:28;
第一段内容

19:45;
第二段内容
```

特征：

- 分隔线使用全角破折号 `—`
- 日期格式 `Y/M/D`
- 时间格式 `HH:MM;`
- 保留原多行内容
- 明文导出后提醒及时删除

## 导入 —— `importDiary(store, password)`

支持两种来源：

1. `data/import_diary.txt`
2. 直接粘贴，输入 `:END` 结束

### 当前解析规则

- 按破折号分隔块
- 识别日期行 `Y/M/D`
- 识别时间行 `HH:MM;`
- 内容读到下一个时间行或文本块末尾
- 兼容全角冒号 / 分号

### 冲突处理

| 选项 | 行为 |
|------|------|
| 完全替换 | 清空当前库，用导入数据替换 |
| 同日覆盖 | 导入数据替换同日期旧数据 |
| 同日保留 | 追加导入 segments 到已有记录后 |

### 导入导出外层页面

现在外层入口页是固定壳子：`TRANSFER.NODE`

底部状态：

- 灰色提示：`Enter执行  Esc返回`
- `>> TRANSFER READY`

注意：

- 目前只重做了外层入口页
- 内层导入 / 导出核心逻辑仍沿用旧实现

## 修改密码 —— `changePasswordInteractive(currentPass)`

流程：

1. 输入当前密码确认身份
2. 输入新密码
3. 再次确认新密码
4. 用旧密码解密，再用新密码重新加密保存
5. 擦除内存中的密码

当前行为：

- 固定壳子页
- `Esc` 取消时直接返回，不再额外弹 `pauseScreen()`
- 不再出现底部“按任意键继续...”把边框吞掉的问题

---

## 神秘计数器 —— `counterPage(store, password)`

页面标识：`COUNTER.NODE`

### 当前能力

- 显示当前计数
- `1. 次数加一`
- `2. 手动设置次数`
- 每次变更后立即自动保存
- 每次变更都会往 `counter_history` 追加一条时间戳
- 右侧面板保留“最近触发记录”标题与分隔线
- 右侧时间链显示格式为 `#编号 : 时间`
- 手动把次数改小后，会先删除超出新编号范围的最近历史，再写入这次新的时间点

### 页面结构

- 左侧：当前计数大字显示
- 右侧：`最近触发记录` 标题 + 分隔线 + 最近几次触发时间链
- 下方：操作菜单

底部状态：

- 灰色提示：`Enter执行  Esc返回`
- `>> COUNTER READY`

## 首次设置 —— `firstTimeSetup()`

首次运行时：

1. 输入密码
2. 再次确认密码
3. 初始化空白 DiaryStore
4. 保存成加密 `diary.enc`

## 登录循环 —— `loginLoop()`

流程：

1. 检查 `diary.enc`
2. 不存在则进入首次设置
3. 最多 3 次密码尝试
4. 成功后进入主循环
5. 连续失败 3 次后退出

---

## 主循环 —— `mainLoop(store, password)`

每次回到主页面时：

1. 计算 `DiaryMetrics`
2. 渲染主页面
3. 在菜单框区域处理主菜单交互
4. 分发到对应功能页

7 个主菜单项：

| 编号 | 选项 | 保存策略 |
|------|------|---------|
| 1 | 写入 / 编辑今日 | 内部自动保存 |
| 2 | 查看全部 | 不改数据 |
| 3 | 按日期编辑 | 内部自动保存 |
| 4 | 导出 / 导入 | 导入时自动保存 |
| 5 | 修改密码 | 内部自动保存 |
| 6 | 神秘计数器 | 内部自动保存 |
| 7 | 保存并退出 | 保存后退出 |

---

## 自动保存策略

| 操作 | 行为 |
|------|------|
| 写入 / 编辑今日 | 确认后立即保存 |
| 按日期编辑（改 / 加 / 删） | 每次操作后立即保存 |
| 导入日记 | 完成后立即保存 |
| 修改密码 | 完成后立即保存 |
| 神秘计数器 | 每次变更后立即保存 |
| 查看全部 / 导出 | 不改数据，不保存 |

---

## 错误处理

| 场景 | 处理 |
|------|------|
| 密码错误 | 解密失败，最多 3 次 |
| 文件不存在 | 进入首次设置 |
| 文件被篡改 | 标签校验失败，等同解密失败 |
| JSON 损坏 | `DiaryStore::load()` 返回 false |
| 导入格式不对 | 提示无法解析，不改数据 |
| 控制台窗口太小 | 固定壳子退化到最小安全尺寸，自适应页重排 |
| resize 期间页面错位 | 交由 `waitForConsoleInputOrResize()` 触发重建 |
