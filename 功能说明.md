# 日记本 - 功能说明

## crypto.h / crypto.cpp —— 加密引擎

直接复用密码库（MiMaKu_Code）的加密代码，仅修改了 MAGIC 标识。

### 常量

| 常量 | 值 | 说明 |
|------|-----|------|
| `MAGIC` | `0x31524944` | "DIR1" little-endian，日记文件标识 |
| `VERSION` | `1` | 文件格式版本号 |
| `HEADER_SIZE` | `48` | 头部长: magic(4)+version(4)+salt(16)+nonce(24) |
| `SALT_LEN` | `16` | Argon2id 盐长度 |
| `NONCE_LEN` | `24` | XChaCha20 Nonce 长度 |
| `KEY_LEN` | `32` | 派生密钥长度 (256-bit) |
| `TAG_LEN` | `16` | 认证标签长度 |
| `ARGON_OPS` | `MODERATE` | Argon2id 操作数 (~3轮) |
| `ARGON_MEM` | `MODERATE` | Argon2id 内存 (~256MB) |

### `bool init()`
调用 `sodium_init()`。必须在使用任何 libsodium 函数前调用。`>= 0` 成功。

### `bool deriveKey(password, passLen, salt, keyOut)`
用 Argon2id 从密码+盐派生 32B 密钥。算法 `crypto_pwhash_ALG_ARGON2ID13`。

### `vector<unsigned char> encryptFile(plaintext, password)`
明文 → `.enc` 二进制。流程：
1. `randombytes_buf` 生成 16B Salt + 24B Nonce
2. `deriveKey` 派生密钥
3. `crypto_aead_xchacha20poly1305_ietf_encrypt` 加密（输出含 16B Tag）
4. 组装：magic(4B) + version(4B) + salt(16B) + nonce(24B) + ciphertext+tag
5. `sodium_memzero` 清密钥

### `optional<string> decryptFile(data, size, password)`
`.enc` 二进制 → 明文 JSON 字符串。流程：
1. 校验 magic/version，提取 salt/nonce
2. `deriveKey` 派生密钥
3. `crypto_aead_xchacha20poly1305_ietf_decrypt` 解密+验证 Tag
4. 失败 → `nullopt`（密码错/文件损坏）；成功 → JSON 字符串

## diary_store.h / diary_store.cpp —— 日记数据管理

封装日记数据的 CRUD 操作，内部使用 JSON 存储。

### JSON 数据结构

```json
{
  "version": 1,
  "entries": [
    {
      "year": 2026,
      "month": 5,
      "day": 1,
      "segments": [
        {"time": "16:56", "content": "早上写的日记"},
        {"time": "19:45", "content": "晚上补充的"}
      ]
    }
  ],
  "counter": 0
}
```

- 一个 entry = 一天的日记
- 一天可以有多个 segments = 多个时间段记录
- 每天只有一篇 entry，不允许多个 entry 同一天
- `counter` 字段用于神秘计数器

### `bool load(filePath, password)`
二进制读取 `.enc` → `crypto::decryptFile` 解密 → `nlohmann::json::parse`。任一失败返回 false。

### `bool save(filePath, password) const`
`data_.dump(2)` 序列化 JSON（缩进 2 空格） → `crypto::encryptFile` 加密 → 二进制写入。

### `void initEmpty()`
初始化 `{"version":1, "entries":[], "counter":0}`，loaded_=true。

### `int findEntry(year, month, day) const`
遍历 entries 查找匹配年月日的条目。返回索引，未找到返回 -1。

### 数据操作
- `entries()` → 返回 entries 数组引用
- `entryCount()` → `entries().size()`
- `addEntry(json)` → `push_back`
- `removeEntry(index)` → 指定索引删除
- `updateEntry(index, entry)` → 替换

### `vector<size_t> getSortedIndices() const`
返回按日期升序排列的条目索引列表（先按年，再按月，再按日）。

### 神秘计数器
- `getCounter()` → 返回 counter 值
- `setCounter(value)` → 设置 counter 值

## page_main.h / page_main.cpp —— 主页面

组装并渲染整个主页面布局。

### `struct MainPageLayout`

| 字段 | 说明 |
|------|------|
| `menuX/Y/W/H` | 菜单区域坐标（interior，不含框线），供 `menuSelectInRegion` 使用 |

### `MainPageLayout renderMainPage(const DiaryMetrics& m, const char* dataPath)`

渲染整个主页面。流程：
1. 获取控制台大小，计算外框尺寸
2. `clearScreen` + `fillRegion` 清空全屏
3. `drawMainFrame` 画外框 `╔═╗║╚╝` + 标题分隔线 `╠═╣`
4. 标题栏：`[>_ SYS.TERMINAL // LOCAL_DIARY_ENV` + `[■][O][X]`
5. 左侧 DIARY ASCII Art 标题 (y=4~9)
6. 右侧 VAULT_INTEGRITY 面板 (y=4~9，和 DIARY 同高 6 行)：
   - 标签 + 分隔线
   - PATH（`dataPath` 实际路径）
   - SIZE（`.enc` 文件大小，动态读取）
   - STATE（`XChaCha20-Poly1305 [LOCKED]`）
   - SYNC（最后保存时间，动态读取）
7. 左侧功能框 + 右侧 MAY ACTIVITY LOG 面板（都落在**限宽居中内容画布**内）：
   - 当月标签 + 分隔线
   - WEEK 1~5 热力图行（`·░▒▓█` 5 级密度）
   - 分隔线 + TOTAL/STREAK/AVG 统计行
8. 右侧 TEMPORAL_METRICS 面板（ACTIVITY LOG 下方隔 1 行）：
   - 标签 + 分隔线
   - DAY PROG 进度条
   - YEAR PROG 进度条
   - WEEK LOG 周打卡 `●○`
9. 底部轻量状态栏（窗口高度足够时显示）：
   - `UPTIME X.Xs`
   - `STATUS IDLE`
10. 命令栏：`>> AWAITING_COMMAND...`

### 自适应策略

- **宽度**：最小 88 列，小于时强制 88
- **内容画布**：在全屏外框内使用约 132 列的限宽居中内容区，避免窗口超宽时主内容被拉散
- **高度**：使用实际窗口高度，**不强制最小高度**——避免小窗口时写入越界导致内容滚动错位
- 各区块通过 `if` 条件检查空间：空间不足时自动跳过（VAULT 优先保留，TEMPORAL 其次，日志栏最后隐藏）

## main.cpp —— 主程序

所有登录、输入交互、编辑器、导入导出、密码管理逻辑均在此文件中。

### 全局控制台句柄

```cpp
extern HANDLE g_hOut = nullptr;  // 标准输出句柄 (定义在 ui_render.cpp)
extern HANDLE g_hIn  = nullptr;  // 标准输入句柄 (定义在 ui_render.cpp)
```

这些全局句柄由 `ui_render.h` 声明，被 `wprint`/`wprintln` 辅助函数和编辑器直接使用。

### 编码与字符工具

编码转换函数（`utf8_to_wstring`、`wstring_to_utf8`、`wcharWidth`）和控制台输出函数（`wprint`、`wprintln`、`clearScreen`）已**迁移至 `ui_render.cpp`**，通过 `ui_render.h` 暴露给所有模块。

### 控制台基础函数

#### `pauseScreen()`
固定写到底部状态行的“按任意键继续...”，等待任意按键后再清掉该行，避免多次暂停时在左下角累积脏输出。

#### `readPassword(prompt)`
复古终端风登录页密码输入：不回显明文，只显示 `*`；输入开始前会显式把光标放回 `[ PASSWORD_ENTRY ]` 中的 `>>` 后方。

#### `readLoginPassword(failedAttempts, maxAttempts)`
登录循环专用密码输入页：
- 第 1 次尝试：密码框亮白
- 第 2 次尝试：密码框黄色警示
- 第 3 次尝试：密码框红色警示
- 错误次数通过框体颜色和状态面板提示表达，不再显示 `(3次机会)` 文本

#### `readLine(prompt, allowEmpty)`
`std::getline` 封装。不允许空输入时可设 `allowEmpty=true`。

#### `readMultiLine(prompt)`
多行粘贴输入。用户粘贴多行文本后，在单独一行输入 `:END` 结束。返回合并的文本。

### 方向键菜单 —— `menuSelect(items, startIdx)`

全屏方向键菜单，用于子页面（按日期编辑、导出/导入、神秘计数器）。实现位于 `ui_render.cpp`。

#### `struct MenuItem`
```cpp
struct MenuItem {
    std::wstring text;   // 显示文本
    bool selectable;     // 是否可选（=false 时为标题/分隔行）
};
```

#### 菜单渲染
- **首次渲染**：`FillConsoleOutputCharacterW` 全屏清空 → `FillConsoleOutputAttribute` 重置属性 → 逐行 `drawLineW`
- **方向键移动**：只重绘两行（旧高亮恢复普通色 + 新高亮反色 0x70），无闪烁
- **输出**：使用 `WriteConsoleW` 直接写控制台

#### 输入处理
- `_getch()` 读键
- `0xE0/0x00 + 0x48` = ↑ 上
- `0xE0/0x00 + 0x50` = ↓ 下
- `\r` = Enter 确认
- 数字键 `0-9` → 匹配菜单项文本开头数字（如 "1. 写入" → 按 1 跳转）
- 循环：到顶按↑跳到底，到底按↓跳到顶
- 返回 `items` 中被选中项的索引

### 区域方向键菜单 —— `menuSelectInRegion(x, y, w, h, items, startIdx)`

在主页面功能框区域内渲染菜单，不破坏周围静态内容。返回逻辑与 `menuSelect` 一致。

- 只重绘高亮变化的两行，避免主页面菜单闪烁
- 进入时隐藏控制台光标，返回前恢复，防止菜单尾部出现可输入光标

### 多行日记编辑器 —— `openDiaryEditor(initialContent)`

命令行的完整文本编辑器，支持多行输入和方向键操作。

#### 返回结构
```cpp
struct EditorResult {
    bool confirmed;       // true=用户确认，false=用户取消(Esc)
    std::wstring content; // 编辑后的内容
};
```

#### 输入模式
- `ReadConsoleInputW` 逐事件读取 `INPUT_RECORD`
- `bKeyDown` 过滤，只处理按键按下事件

#### 支持的操作

| 按键 | 行为 |
|------|------|
| 普通字符 / 中文 | 在光标处插入字符 |
| Shift+Enter / Ctrl+Enter | 插入换行符 `\n` |
| 普通 Enter | 进入**确认模式**（底部提示栏） |
| 确认模式中 Enter | 返回 confirmed=true |
| 确认模式中 Esc | 退回继续编辑 |
| Esc（编辑模式中） | 取消，返回 confirmed=false |
| ↑↓←→ | 移动光标 |
| Home / End | 行首 / 行尾 |
| Backspace / Delete | 删除光标前/后字符 |

#### 渲染机制
- 内部使用 `std::wstring` 缓冲区
- `calcEditorLines(buf, screenW)` 计算行信息（处理折行和中文字符宽度）
- `findCursorPos()` 定位光标行号列号
- `rowColToIndex()` 将行列转回 buffer 索引
- 为 `TODAY_LOG` 和 `INPUT` 各维护“上一帧可见行缓存”，仅对变化行执行 `writeAtColor`，降低持续输入时的闪烁
- `SetConsoleCursorPosition` 精确定位光标

#### 确认模式提示栏
```
Enter=确认保存    Esc=继续编辑
```
反色高亮显示在屏幕最底部，进入确认模式后只接受 Enter 和 Esc。

### 写入 / 编辑今日 —— `writeOrEditToday(store, password)`

日记本核心操作。

**流程**：
1. 获取当前日期和时间
2. 调用 `findEntry()` 查找今日日记
3. **如果没有** → 创建新 entry，segments 为空 → 添加一个空 segment（当前时间）
4. **如果有** → 备份原 entry → 添加一个空 segment（当前时间）
5. 组装 `TODAY_LOG`：显示今天已有记录，并为当前时间片准备实时预览槽位
6. 调用 `openDiaryEditor("")` 让用户编辑
7. **用户确认且有内容** → 更新 segment 内容，保存
8. **用户确认但内容为空** → 删除空 segment（如 entry 为空则删除整篇），保存
9. **用户取消(Esc)** → 恢复备份（已有日记）或删除新 entry（新建日记）

**效果**：
- 今天第一次写 → 创建新日记，自动补充当前时间点
- 今天已有日记 → 打开已有日记，自动补充新时间点，续写内容
- 页面布局为 **大 `TODAY_LOG` + 小 `INPUT`**：上方主视图负责阅读今天完整日志流，下方输入区只负责打字
- 当前输入内容会实时投影到 `TODAY_LOG` 末尾，形成“今天完整日志流”预览
- `Alt+Up/Down` 可滚动 `TODAY_LOG` 历史视图
- 取消 → 自动清理空时间点，不留痕迹

### 按日期编辑 —— `editByDate(store, password)`

三级子菜单逐级选择：

**选择年份**：
- 从所有 entries 提取不重复的年份，按数值排序
- 方向键选择或数字键跳转

**选择月份**：
- 筛选选中年份下的月份列表
- 方向键选择或数字键跳转

**选择日期**：
- 筛选选中年月下的日期列表
- 方向键选择或数字键跳转

**日期操作界面**：
| 菜单项 | 功能 |
|--------|------|
| 查看日记（只读） | 以排版格式展示全部内容，按任意键返回 |
| 编辑记录 N | 打开编辑器修改某条 segment |
| 添加新记录 | 自动补充当前时间，打开编辑器写新 segment |
| 删除某条记录 | 输入编号 + yes 确认，删除后如为空则删除整篇 |
| 删除整篇日记 | yes 确认后删除整篇 entry |
| 0. 返回 | 回到上层 |

### 导出 —— `exportDiary(store)`

将全部日记导出为明文 `data/export_diary.txt`。

**导出格式**：严格按衫先生指定的排版：
```
————————————————————————
2026/3/30

17:28;
日记内容第一段

19:45;
日记内容第二段

————————————————————————
2026/4/3

12:00;
另一天的日记

```

- 分隔线使用 24 个全角破折号 `—`（UTF-8: E2 80 94）
- 日期格式 `Y/M/D`（无前导零）
- 年月日与时间之间有一个空行
- 时间格式 `HH:MM;`（分号结尾）
- 日记内容直接拼接，保持多行格式
- 导出后提示 "该文件是明文，用完后请及时删除"

### 导入 —— `importDiary(store, password)`

支持两种导入方式：
1. **从文件导入**：读取 `data/import_diary.txt`
2. **直接粘贴导入**：用户粘贴文本后输入 `:END` 结束

#### 导入解析器

解析算法：
1. 按分隔线（连续全角破折号 `——`）分割文本块
2. 每个文本块解析一个 entry：
   - 寻找日期行（含 2 个 `/` 符号）：`YYYY/M/D` 或 `YYYY/MM/DD`
   - 寻找时间行（含冒号且以分号结尾）：`HH:MM;` 或 `HH：MM；`
   - 时间行下方的内容直到下一个时间行或文本块结束
3. 兼容处理：
   - 全角冒号 `：` → 自动转为半角 `:`
   - 全角分号 `；` → 自动去除
   - 末尾空行自动修剪
   - 连续的 `—` 行自动跳过

#### 导入冲突处理

| 选项 | 行为 |
|------|------|
| 1. 完全替换 | 清空当前日记库，用导入数据替换 |
| 2. 合并，同日覆盖 | 同日期时用导入数据替换原有数据 |
| 3. 合并，同日保留 | 同日期时把导入的 segments 追加到已有 segments 后面 |
| 0. 取消导入 | 不作任何改动 |

### 修改密码 —— `changePasswordInteractive(currentPass)`

流程：
1. 验证当前密码
2. 输入新密码 + 确认
3. 用旧密码解密 `diary.enc` → 用新密码加密保存
4. `sodium_memzero` 清理内存中的密码

### 神秘计数器 —— `counterPage(store, password)`

内置计数器功能，数据存储在 `diary.enc` 的 `counter` 字段中。

- 显示当前计数
- 次数加一（每次操作自动保存）
- 手动设置次数

### 首次设置 —— `firstTimeSetup()`

首次运行时引导：
1. 显示欢迎信息
2. 设置日记密码 + 确认
3. 创建空白日记库 → 加密保存 `diary.enc`
4. `sodium_memzero` 清密码 → 提示重新运行登录

### 登录循环 —— `loginLoop()`

1. 检查 `diary.enc` 是否存在 → 不存在则进 `firstTimeSetup()`
2. 最多 3 次输入机会
3. 每次失败后显示 `ACCESS WARNING` 状态页，再继续下一次尝试
4. 密码正确 → 进入 `mainLoop()`
5. 密码错误 3 次 → 显示 `ACCESS DENIED` 后退出程序

### 主循环 —— `mainLoop(store, password)`

每次进入主页面时：
1. 调用 `computeMetrics(store)` 计算全部统计指标
2. 调用 `renderMainPage(m, DIARY_PATH)` 渲染主页面
3. 调用 `menuSelectInRegion()` 在功能框内处理菜单交互
4. 根据选择执行对应操作（写入/查看/编辑/导出导入/密码/计数器/退出）

主页面现行为：
- 使用“全屏外框 + 限宽居中内容画布”，超宽窗口时内容主体不再被横向拉散
- 左下角只保留两行轻量状态：`UPTIME` 与 `STATUS IDLE`
- `>> AWAITING_COMMAND...` 是主页面安全光标落点

7 个主菜单项：

| 编号 | 选项 | 保存策略 |
|------|------|---------|
| 1 | 写入 / 编辑今日 | 内部自动保存 |
| 2 | 查看全部 | 不改数据，不保存 |
| 3 | 按日期编辑 | 内部自动保存 |
| 4 | 导出 / 导入 | 内部自动保存（导入时） |
| 5 | 修改密码 | 内部自动保存 |
| 6 | 神秘计数器 | 内部自动保存 |
| 7 | 保存并退出 | 保存后退出 |

### main() 入口

1. `SetConsoleOutputCP(CP_UTF8)` + `SetConsoleCP(CP_UTF8)` 控制台 UTF-8
2. 获取 `g_hOut` / `g_hIn` 全局句柄
3. 记录 `g_startTick = GetTickCount()` 启动时间
4. `crypto::init()` 初始化 libsodium
5. `CreateDirectoryA("data")` 确保 data 目录
6. `loginLoop()` 进入登录流程

## 自动保存策略

| 操作 | 行为 |
|------|------|
| 写入 / 编辑今日 | 确认后立即自动保存 |
| 按日期编辑（编辑/添加/删除） | 每次操作后自动保存 |
| 导入日记 | 操作完成后自动保存 |
| 修改密码 | 内部自动保存 |
| 神秘计数器 | 每次操作后自动保存 |
| 7. 保存并退出 | 保存后退出 |
| 查看全部 / 导出 | 不改数据，不保存 |

## 错误处理

| 场景 | 处理 |
|------|------|
| 密码错误 | 解密 Tag 不匹配 → `nullopt`，最多 3 次 |
| diary.enc 不存在 | 进 `firstTimeSetup()` 首次引导 |
| diary.enc 被篡改 | Tag 不匹配 → 解密失败返回密码错误 |
| data 目录不存在 | `main()` 中 `CreateDirectoryA` 自动创建 |
| JSON 损坏 | `DiaryStore::load()` 捕获 json 异常，返回 false |
| 导入格式不正确 | 解析器输出提示"未能解析"，不做任何改动 |
| libsodium 初始化失败 | 输出错误，暂停后退出 |
| 方向键菜单无可用项 | `menuSelect` 返回 -1（理论上不会出现） |
| 编辑器内容过大 | 屏幕高度不足时自动滚动 |
| 窗口高度不足 | 各区块通过 `if` 条件自动跳过（VAULT → TEMPORAL → 日志栏），不导致滚动错位 |
