# 日记本 - 功能说明

## crypto.h / crypto.cpp —— 加密引擎

### 常量

| 常量 | 值 | 说明 |
|------|-----|------|
| `MAGIC` | `0x31524944` | `DIR1` little-endian，日记文件标识 |
| `VERSION` | `1` | 文件格式版本号 |
| `HEADER_SIZE` | `48` | `magic + version + salt + nonce` |
| `SALT_LEN` | `16` | Argon2id 盐长度 |
| `NONCE_LEN` | `24` | XChaCha20 Nonce 长度 |
| `KEY_LEN` | `32` | 派生密钥长度 |
| `TAG_LEN` | `16` | 认证标签长度 |

### `bool init()`

调用 `sodium_init()`，初始化 libsodium。

### `vector<unsigned char> encryptFile(plaintext, password)`

流程：

1. 生成随机 Salt + Nonce
2. 用 Argon2id 从密码派生密钥
3. 用 XChaCha20-Poly1305 加密
4. 组装二进制头 + 密文
5. 擦除密钥内存

### `optional<string> decryptFile(data, size, password)`

流程：

1. 校验 magic/version
2. 提取 Salt / Nonce
3. 派生密钥
4. 解密并验证认证标签
5. 成功返回 JSON 字符串，失败返回 `nullopt`

## diary_store.h / diary_store.cpp —— 日记数据管理

负责 JSON 数据、按日期 CRUD、神秘计数器与历史时间链、加密读写。

### JSON 数据结构

```json
{
  "version": 1,
  "entries": [
    {
      "year": 2026,
      "month": 5,
      "day": 1,
      "segments": [
        {"time": "16:56", "content": "早上写的日记"},
        {"time": "19:45", "content": "晚上补充的"}
      ]
    }
  ],
  "counter": 0,
  "counter_history": []
}
```

- `entries`：按天存日记
- `segments`：同一天内多条时间段记录
- `counter`：神秘计数器当前值
 - `counter_history`：触发计数器的时间戳数组
  - 仍然只存时间戳，不单独存编号
  - 计数器页面右侧显示的 `#编号` 由当前 `counter` 在渲染时倒推得到
  - 整体历史回望会按年/月浏览这份完整历史，并支持删除单条记录

### 核心接口

| 接口 | 说明 |
|------|------|
| `load(filePath, password)` | 解密 `.enc` 并解析 JSON |
| `save(filePath, password) const` | JSON 序列化后重新加密写回 |
| `initEmpty()` | 初始化空白库，含 `counter` 与 `counter_history` |
| `findEntry(year, month, day)` | 按日期找条目 |
| `addEntry(entry)` | 添加新条目 |
| `removeEntry(index)` | 删除条目 |
| `updateEntry(index, entry)` | 替换条目 |
| `getSortedIndices()` | 返回按日期升序的索引列表 |
| `getCounter()` / `setCounter()` | 读取 / 修改计数器 |
| `getCounterHistory()` | 读取最近触发时间链 |
| `pushCounterHistory(timestamp, maxCount)` | 追加一条时间戳；默认不裁剪，传上限时才裁剪 |
| `trimCounterHistoryTail(removeCount)` | 从历史尾部删掉最近几条触发记录 |
| `removeCounterHistoryAt(index)` | 按索引删除一条计数器历史 |

### 布局坐标系约定

自适应页面的布局计算**必须使用视口尺寸（`srWindow`）**，禁止使用缓冲区尺寸（`dwSize`）：

```cpp
// 正确：用 srWindow 计算可见区域
int w = csbi.srWindow.Right - csbi.srWindow.Left + 1;
int h = csbi.srWindow.Bottom - csbi.srWindow.Top + 1;

// 错误：dwSize 包含了不可见的历史回滚区域
int w = csbi.dwSize.X;
int h = csbi.dwSize.Y;
```

原因：传统 cmd.exe 默认缓冲区高度为 9001 行，远大于可见窗口。若用 `dwSize.Y` 计算布局，
所有绘制操作将溢出视口，导致滚动条剧烈变化或 resize 死循环。

相关：`getConsoleViewport()` 已封装此逻辑，固定壳子页面一直使用它。自适应页面内部的
`renderShell()` / `renderEditorShell()` / `rebuildLayout()` 现已统一为视口尺寸。

## 写入 / 编辑今日 —— `writeOrEditToday(store, password, diaryPath)`

相关文件：

- `src/page_today.h`
- `src/page_today.cpp`
- `src/diary_editor.h`
- `src/diary_editor.cpp`

这是日记本最核心的交互。

### 当前行为

1. 获取当前日期和时间
2. 查找今日日记
3. 如果没有则新建今天的 entry
4. 如果已有则给今天追加一个空 segment
5. 组装 `TODAY_LOG`
6. 打开多行编辑器 `openDiaryEditor()`
7. 用户确认后保存；取消则回滚

说明：

- 该功能已从 `main.cpp` 拆到 `page_today.*`
- 编辑器核心逻辑已独立到 `diary_editor.*`
- `writeOrEditToday()` 现在通过参数接收保存路径，不再直接使用 `main.cpp` 的静态常量

### 编辑器特征

- 上方：`TODAY_LOG`
- 下方：`INPUT`
- 当前输入会实时投影到 `TODAY_LOG` 末尾
- `PgUp/PgDn` 翻历史页
- `Alt+Up/Down` 细滚动历史视图
- `Up/Down` 保留给编辑光标移动，不与阅读页统一
- 中文输入法下的弯引号 `“ ” ‘ ’` 已按单宽字符处理，不再出现“实际输入一个字符、光标视觉多空一格”的错位
- 按 `Enter` 显示 `保存编辑?  Enter=确认保存    Esc=继续编辑`
- `INPUT` 非空时按 `Esc` 不会直接退出，而是显示 `退出编辑?  Enter=确认保存    Esc=继续编辑`
- `INPUT` 为空时按 `Esc` 仍直接取消

## 神秘计数器 —— `counterPage(store, password, diaryPath)`

相关文件：

- `src/page_counter.h`
- `src/page_counter.cpp`

页面标识：`COUNTER.NODE`

### 当前能力

- 显示当前计数
- 当前计数数字使用与主页面 `DIARY` 同语系的 6 行高大字
- 大字数字的测宽与绘制基于自动扫描的真实笔画边界，不依赖手工填写左右留白参数
- `1. 次数加一`
- `2. 手动设置次数`
- `3. 整体历史回望`
- 每次变更后立即自动保存
- 每次变更都会往 `counter_history` 追加一条时间戳
- 右侧面板保留“最近触发记录”标题与分隔线
- 右侧时间链显示格式为 `#编号 : 时间`
- 手动把次数改小后，会先删除超出新编号范围的最近历史，再写入这次新的时间点
- 进入整体历史回望后，先选年份，再选月份，再看该月全部触发记录
- 在历史回望里删除单条记录后，计数器会同步减 1

### 页面结构

## 首次设置 —— `firstTimeSetup()`

首次运行时：

1. 输入密码
2. 再次确认密码
3. 初始化空白 DiaryStore
4. 保存成加密 `diary.enc`

---

## 登录循环 —— `loginLoop()`

流程：

1. 检查 `diary.enc`
2. 不存在则进入首次设置
3. 最多 3 次密码尝试
4. 成功后进入主循环
5. 连续失败 3 次后退出

---

## 主循环 —— `mainLoop(store, password)`

每次回到主页面时：

1. 计算 `DiaryMetrics`
2. 进入主页面会话并首绘页面
3. 在 `page_main.*` 内部处理主菜单交互与空闲局部刷新
4. 返回选择结果并分发到对应功能页

7 个主菜单项：

| 编号 | 选项 | 保存策略 |
|------|------|---------|
| 1 | 写入 / 编辑今日 | 内部自动保存 |
| 2 | 查看全部 | 不改数据 |
| 3 | 按日期编辑 | 内部自动保存 |
| 4 | 导出 / 导入 | 导入时自动保存 |
| 5 | 修改密码 | 内部自动保存 |
| 6 | 神秘计数器 | 内部自动保存 |
| 7 | 保存并退出 | 保存后退出 |

---

## 自动保存策略

| 操作 | 行为 |
|------|------|
| 写入 / 编辑今日 | 确认后立即保存 |
| 按日期编辑（改 / 加 / 删） | 每次操作后立即保存 |
| 导入日记 | 完成后立即保存 |
| 修改密码 | 完成后立即保存 |
| 神秘计数器 | 每次变更后立即保存 |
| 查看全部 / 导出 | 不改数据，不保存 |

## 错误处理

| 场景 | 处理 |
|------|------|
| 密码错误 | 解密失败，最多 3 次 |
| 文件不存在 | 进入首次设置 |
| 文件被篡改 | 标签校验失败，等同解密失败 |
| JSON 损坏 | `DiaryStore::load()` 返回 false |
| 导入格式不对 | 提示无法解析，不改数据 |
| 控制台窗口太小 | 固定壳子退化到最小安全尺寸，自适应页重排 |
| resize 期间页面错位 | 交由 `waitForConsoleInputOrResize()` 触发重建 |
